# Security Policy

## Supported Versions

Security updates are applied to the `main` branch. Please base security-related
patches on `main` unless otherwise coordinated with maintainers.

## Reporting a Vulnerability

We take security seriously and appreciate responsible disclosure.

- Preferred: Open a private advisory via GitHub Security Advisories for this
  repository.
- Alternative: Email security reports to security@your-org.example.

Please include:
- A description of the issue and potential impact
- Steps to reproduce or a proof-of-concept
- Any relevant logs, configurations, or environment details

We ask that you do not open public issues for security vulnerabilities.

## Disclosure Timeline

- Acknowledgment of receipt: within 3 business days
- Initial assessment and triage: within 7 business days
- Coordinated fix and release timeline: determined by severity and impact; we
  aim to resolve high-severity issues within 90 days

We will credit reporters in release notes, unless you prefer to remain
anonymous.
